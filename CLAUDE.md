# 프로젝트 규칙 (매 작업 전 반드시 확인)

이 프로젝트에서 작업할 때 아래 규칙을 항상 지켜주세요. 이걸 몰라서 반복적으로 빠뜨리거나 잘못 구현하는 실수가 있었습니다.

---

## 1. 이미지는 두 시스템으로 완전히 분리되어 있음 — 절대 섞지 말 것

| 용도 | 함수 | 데이터 출처 | 화면 |
|---|---|---|---|
| **게임(정밀검사) 아이콘** | `imgSrc(state, species, stage)` | `IMG` 객체, 6종만 존재 | SpeciesSelectScreen, DayScreen |
| **추천 결과 실제 사진** | `realPhoto(species)` | `REAL_PHOTOS` 객체, 21종 존재 | HabitResultScreen, ReportScreen, RecommendationResultScreen |

**어떤 화면을 만들든, 아래 표로 어떤 함수를 써야 할지 먼저 확인할 것:**
- 21일 게임 화면(식물 선택, 매일 화분) → `imgSrc()`, 게임용 6종만 (`GAME_SPECIES_IDS`)
- 추천/결과/매칭 화면(카드로 식물 보여줄 때) → 반드시 `realPhoto()`, 21종 전체 (`PLANTS`)

**컴포넌트를 새로 만들거나 수정할 때마다, 식물 이미지를 보여주는 부분이 있으면 이 표에 맞는 함수를 쓰고 있는지 확인할 것.** 지금까지 이걸 빠뜨려서 "이미지가 아예 안 나옴" 버그가 반복됐음.

---

## 2. "구매하기" 버튼은 쿠팡 검색 링크가 정상 동작임 (버그 아님)

```javascript
window.open(`https://www.coupang.com/np/search?q=${encodeURIComponent(name+' 화분')}`, '_blank')
```

이건 **의도된 설계**입니다. 실제 상품 페이지로 바로 연결하려면 쿠팡파트너스 제휴 API 승인(사업자 등록 필요)이 있어야 하는데, 지금 프로토타입 단계에서는 불가능합니다. **"검색 결과로 이동"이 최종 사양**이며, 고치거나 "더 나은 링크로 바꿔야 하나" 고민할 필요 없습니다.

---

## 3. 게임용 식물(6종)과 추천용 식물(21종)은 다른 배열임

```javascript
const GAME_SPECIES_IDS = ['stucky','zz','echeveria','pothos','philodendron','monstera'];
const GAME_PLANTS = PLANTS.filter(p => GAME_SPECIES_IDS.includes(p.id));
```

- **게임 화면(식물 선택, 21일 시뮬레이션)**: 반드시 `GAME_PLANTS`(6종)만 사용. 21종 전체를 보여주면 게임용 아이콘이 없는 15종은 이미지가 깨짐.
- **추천/매칭 로직(빠른진단, 정밀검사 리포트, 식물추천받기)**: `PLANTS`(21종) 전체 사용.

---

## 4. 확정적 진단 표현 금지 — 항상 신뢰도/관측횟수 명시

리포트나 성향분석 화면에서 "당신은 OO형입니다"처럼 단정하지 말 것. 반드시 다음을 포함:
- 관측 횟수(급수 몇 회, 이상반응 체크 몇 회)
- 신뢰도 문구 (`totalSlots >= 30` → "꽤 일관된 패턴", `>= 15` → "어느 정도 참고할 만함", 미만 → "참고 정도로만")
- "확정된 진단이 아니라 참고 자료" 같은 면책 문구

---

## 5. 상태 체크를 안 하고 넘어가면 감점되어야 함 (핵심 밸런스 로직)

이상 상태(반점/벌레/마름)가 떠 있는데 유저가 확인 버튼을 아예 안 누르고 "다음 날"로 넘어가면, **오답과 동일하게 감점**되어야 합니다 (`-15`). 이게 없으면 "아무것도 안 하는 게 제일 안전한 전략"이 되어버리는 버그가 생깁니다. `nextDay()` 함수를 만들거나 수정할 때 항상 이 로직이 살아있는지 확인할 것.

**현재 정확한 수치:**
- 이상 이벤트 발생 확률: 45% (하루마다, 이미 이상이 없을 때만)
- 정답이지만 늦게 반응: `(반응일수-1) × 10`점 감점
- 오답 또는 미확인: `-15`점

---

## 7. 코드 참고 시 이미지 없는 로직 전용 파일을 볼 것

`prototype/plant_habit_app.html`(원본, 이미지 base64 포함, 4MB)이 아니라 **`prototype/plant_habit_app_LOGIC_ONLY.html`(이미지 제거, 50KB)을 참고할 것.** 로직·구조는 완전히 동일하고 훨씬 가벼움. 원본은 이미지 데이터가 필요할 때만 예외적으로 열 것.

---

## 8. 급수 시간대 제한 (최신 사양)

- **기본 모드**: 오전 10시~밤 10시(22시)를 슬롯 개수만큼 균등 분할, 그 시간대에만 급수 버튼 활성화
- **치트 모드**: 비밀번호 입력(`DevCheatPanel`)으로 전환, 시간 제한 없이 언제든 급수 + 다음날 이동 가능
- 기본값(초기 상태)은 반드시 **일반 모드**(시간 제한 있음)여야 함. 치트 모드가 기본이 되지 않도록 주의.
