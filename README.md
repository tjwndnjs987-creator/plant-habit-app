## 지금 프로토타입이 하는 일 (전체 기능 요약)

1. **빠른 진단**: 습관 25문항(예/아니오) → 물관리·반응속도·꾸준함 3축 점수 → 21종 중 매칭 추천 (실제 사진 포함)
2. **정밀검사**: 게임용 대표 6종 중 선택 → 물주기 계획(하루 최소 2번, 1회량) 직접 설정 → 21일 진행
   - 매일 화분 이미지(성장 4단계 × 상태 4종) 표시
   - 상태체크(이상없음/반점/마름/벌레), 안 하면 감점
   - 슬롯별 급수(홀드 제스처로 채우기), 안 하면 "건너뜀" 기록
   - 시간 기반 시도 제한 + 광고 시청으로 대기 스킵/부족분 채우기 (데모용 15초, 실배포 시 시간 단위로)
   - 포인트·콤보·업적 배지
   - 21일 후 리포트(3축 점수 + 관측 횟수·신뢰도 문구 + 획득 포인트/배지)
3. **성향분석**: 21일 데이터 기반 유형 라벨(느긋한 방치형 등), 판정 기준 명시
4. **식물 추천받기**: 빛/공간/목적 3문항 + 습관 점수 결합 → 최종 추천 + 구매(쿠팡)/관리앱 버튼

## VS Code에서 첫 프롬프트로 이렇게 시작하세요

```
prototype/plant_habit_app.html은 이미 완성해서 작동을 확인한 프로토타입이야.
이 로직과 기능을 하나도 빼거나 바꾸지 말고, React 프로젝트 구조로 나눠줘.
이미지는 assets/images 폴더에 이미 준비해뒀으니 public 폴더로 옮겨서 참조해줘
(지금 프로토타입은 이미지가 base64로 박혀있는데, 이제 파일 경로로 바꿔줘).
한 번에 다 하지 말고, 먼저 컴포넌트 파일 구조 계획만 보여줘.
```

## 아직 안 끝난 것 — 우선순위대로

| 순위 | 항목 | 상태 |
|---|---|---|
| 1 | **데이터 저장** | 새로고침하면 21일 진행상황이 다 날아감. localStorage 또는 서버 저장 필요 |
| 2 | **13종 사진 임시 대체 중** | 8종(스투키·몬스테라·에케베리아·스킨답서스·금전수·필로덴드론·산세베리아·호접란)만 실사진 확보. 나머지는 코드에 `TODO` 표시된 임시 사진. Unsplash API로 교체 권장: |
```javascript
fetch(`https://api.unsplash.com/search/photos?query=${species}&client_id=YOUR_KEY`)
```
| 3 | **실제 광고 SDK 연동** | 지금은 "광고 재생 중..." 연출뿐. AdMob 등 리워드 광고 SDK로 교체 필요 |
| 4 | **시간 상수 조정** | `TURN_WAIT_MS`(현재 15초 데모용) 등을 실제 배포 값으로 변경 |
| 5 | **정밀검사 탭 이탈 시 초기화 버그** | 탭 전환하면 진행 중이던 21일 게임이 리셋됨 — 1번(데이터 저장)과 함께 해결 필요 |
